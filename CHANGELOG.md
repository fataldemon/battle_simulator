# Changelog

All notable changes to this project will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.0.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [Unreleased]

## [v42.0] - 2026-04-27
### Added
- **参数化技能引擎**：`skill.py` 的 `AttackEffect` 类现支持通过 `params` 动态接收倍率 (`multiplier`)、浮动值 (`variance`) 和暴击率 (`crit_rate`)，解耦了角色属性与技能定义。
- **模块化信任标识**：新增 `skill_module_handled` 返回类型，用于标记已完成自我结算的技能动作（如治疗、Buff），防止主程序二次干预。

### Changed
- **核心架构重构**：彻底移除了 `player.py` 中冗余的手动伤害计算、暴击判定与死亡检测代码。所有战斗数值结算统一收归至 `skill.py` 的 `execute` 方法中执行，实现了真正的“单一事实来源”（Single Source of Truth）。
- **日志输出标准化**：全员的攻击日志格式现已完全统一（如 `💥 爱丽丝 对 铁皮傀儡 造成了 53 点伤害!`），消除了不同角色技能描述参差不齐的杂乱现象。

### Fixed
- **致命崩溃修复**：彻底解决了因小绿治疗技能已在内部完成加血，但主程序仍试图读取不存在的 `action['amount']` 字段而导致的 `KeyError` 崩溃问题。
- **全量死亡提示**：得益于底层架构的统一，现在无论是谁的普攻、技能还是怪物的攻击，只要造成目标死亡，都会正确无误地打印出专属的倒地台词（如 `💀 xxx 倒下了...`）。

## [v41.0] - 2026-04-27
### Added
- **通用死亡检测器**：在 Player 类中增加了 `_check_death()` 静态方法，旨在通过手动调用来补全击杀后的反馈信息。

### Fixed
- **击杀反馈缺失**：初步尝试修复了玩家普攻秒杀敌人时，系统未显示“倒下了”字样和尸体图标的问题。（注：该修补方案已被 v42 的深层重构方案取代）。

## [v40.0] - 2026-04-26
### Added
- **智能索敌系统**：怪物在使用 Debuff/Trap/Stun 等控制技能时，会优先选择身上**没有**该特定负面状态的目标，避免无效叠加。
- **技能自动切换**：当控制技能找不到符合条件的目标时，怪物会自动切换为普通攻击，不再浪费行动回合。
- **全速追击机制**：怪物在无法攻击时，会直接移动到能打到目标的位置（最大移动5格），而不是每次只移动一格。
- **追击后攻击**：怪物移动后会立即在同一回合内尝试再次攻击，实现了类似玩家的"调整位置+攻击"的连贯动作。

### Changed
- **怪物 AI 重构**：重写了 `Monster.decide_action()` 方法，整合了智能索敌、技能切换和全速追击逻辑。
- **移动逻辑优化**：`_move_towards_target()` 现在支持直接移动到指定位置，而不仅仅是逼近一步。

### Fixed
- **Debuff 重复施加问题**：修复了怪物会对已有相同 debuff 的目标重复施加状态的 Bug。
- **追击效率低下**：修复了怪物追击时每次只能移动一格的低效行为。

## [v39.0] - 2026-04-26
### Fixed
- **Debuff 机制修复**：在 `player.py` 和 `monster.py` 的 `update_status_effects` 函数中补全了 `atk_down` 和 `defense_down` 的处理逻辑，确保状态结束后属性能正确恢复。
- **台词格式统一**：将所有角色的技能打印格式统一为双引号 `" "`，并移除了多余的感叹号，保持输出风格一致。
- **技能菜单优化**：调整了爱丽丝的技能选择顺序，现在为 [普攻, 充能, EX]，更符合直觉。

## [v37.0] - 2026-04-25
### Added
- **全语音台词系统 (Quote System)**：重构了对话输出逻辑，所有技能类均新增 `quote` 属性，实现了动态台词播放。
- **全员配音增强**：爱丽丝、队友（桃井、小绿、柚子）以及各类怪物现在都有专属的行动喊话。

### Changed
- **台词输出统一**：移除了 `player.py` 中的硬编码 `print` 语句，改为统一调用技能模块的输出接口。
- **平衡性调整**：修正了近战怪物（如史莱姆、哥布林）的攻击射程至合理范围（Range 1）。
- **优香大魔王重制**：更新了其技能名称与台词，增强了 Boss 战的压迫感。

### Fixed
- **修复柚子复读机 Bug**：解决了柚子前辈在战斗中重复打印技能描述和喊话的问题。

## [v30.1] - 2026-04-25
### Added
- **怪物AI追击逻辑**：实现了 `Monster.chase_target()` 方法。当目标超出射程时，怪物不再跳过回合，而是会主动向玩家逼近，增加了战斗的压迫感！
### Changed
- **战斗日志优化**：重新设计了伤害计算的输出格式，现在更加简洁明了。
- **台词沉浸感升级**：将原本生硬的角色行动名（如 `attack`）替换为更具代入感的描述性台词（如"接招吧！光之剑！"）。
### Fixed
- **语法错误修复**：修正了之前版本中遗留的一个导致控制台报错的缩进 Bug。

## [v30.0] - 2026-04-25
### Added
- **战术系统重构**：引入线性队列 (Linear Queue) 概念，战场由一维数组管理。
- **挤压式换位 (Squeeze Move)**：实现 `utils.py` 中的挤压算法，支持单位自由穿插与推挤。
- **基于射程的索敌**：所有技能均绑定 `range` 属性，实现了 `_find_valid_targets` 逻辑。
- **回合流程拆分**：战斗循环拆分为严格的【移动阶段】→【行动阶段】。
- **战后清扫**：阵亡单位自动离场，剩余人员向前靠拢。
### Changed
- **输入格式升级**：指令格式从"目标序号"变更为"位置编号"（如 `3-1`）。
- **UI 提示优化**：新增战场站位概览与可攻击目标列表显示。
- **技能平衡性**：重新规划了全量技能的射程梯度（近战 1-3，中程 4-6，远程 7-10，全屏 99）。
- **代码结构**：重构了 `main.py`, `player.py`, `monster.py`, `skill.py` 以适应新机制。
