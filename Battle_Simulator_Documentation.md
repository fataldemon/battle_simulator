# ⚔️ 战斗模拟器 v12.0 架构与功能文档

## 1. 系统概览
这是一个基于 Python 开发的模块化回合制战斗模拟器。采用**主程序 + 数据驱动**的设计，支持多名玩家角色对抗具有独特机制的 Boss。
在 v14.0 版本中，系统进行了**台词系统的全面重构**，实现了所有角色行动台词的动态读取，进一步提升了战斗的沉浸感。

## 2. 核心模块架构

### 2.1 主程序 (`team_battle_v14.0_main.py`)
*   **游戏流程**：初始化队伍 -> 随机抽取怪物 -> 进入战斗循环 -> 判定胜负。
*   **战斗循环**：
    1.  **Boss 行动阶段**：执行怪物逻辑（攻击、施法、受控）。
    2.  **结算阶段**：检查血量，判定游戏结束。
    3.  **玩家行动阶段**：依次执行队友（桃井、小绿、柚子）和主角（爱丽丝）的行动。
    4.  **状态更新**：回合结束时，更新所有角色的 Buff/Debuff 状态。
*   **UI 反馈 (v12.0 新增)**：
    *   **可视化血条**：实时打印 HP 进度条（例如：`[████░░░░░░] 50/100`）。
    *   **状态图标**：在血条下方显示当前的增益/减益图标（如 ⚔️ 攻击力提升、💚 治愈）。
    *   **输入提示优化**：爱丽丝行动时，提前显示输入提示符。
*   **日志顺序修复 (v11.4)**：
    *   修复了“死亡信息在受伤信息之前”的 Bug。
    *   现在，`take_damage` 函数不再负责打印死亡信息。
    *   主程序在调用 `take_damage` 并打印伤害信息后，检查目标是否存活。如果不存活，则打印死亡信息。
    *   确保显示顺序为：`💥 击中了 XX，造成 YY 点伤害！(剩余HP: 0)` -> `💀 XX 倒下了...`。

### 2.2 怪物模块 (`monster.py`)
*   **数据结构**：每个怪物拥有独立的属性面板（HP/ATK/LV）和**自定义技能表 (`skills_list`)**。
*   **通用技能库**：包含普攻、重击、自愈、护盾、火焰球、嚎叫、狂暴等基础逻辑。
*   **特色机制 (v12 核心亮点)**：
    *   🕷️ **机械蜘蛛**：【蛛网束缚】—— 强制使一名玩家下回合无法行动。
    *   🦀 **巨型螃蟹**：【缩壳防御】—— 大幅提升防御并反弹部分伤害。
    *   👻 **幽灵猫**：【相位闪烁】—— 100% 闪避下一次攻击。
    *   🪨 **石头巨人**：【地震术】—— 对全场玩家造成群体伤害。
    *   🤖 **铁皮傀儡**：【电磁脉冲】—— 降低全场玩家的防御力。
    *   🐉 **古代巨龙**：【龙息吐息】—— 对全场玩家造成高额火焰伤害。
    *   🌀 **虚空吞噬者·终焉**：【视界崩坏】—— 消除我方增益效果；【生命汲取】—— 造成伤害时回复自身生命值。
    *   📐 **优香大魔王**：【精确计算】—— 30% 几率造成 2.5 倍暴击伤害。

### 2.3 玩家模块 (`player.py`)
*   **爱丽丝 (手动交互)**：
    *   **光啊！**：积攒能量（1次充能=1层）。
        *   *v12 新增说明*：积攒光之能量，为释放终极必杀技做准备。
    *   **EX 技能 (光哟！！！)**：消耗 2 层能量，造成巨额伤害（附带暴击判定）。
        *   *v12 新增说明*：消耗所有光之能量，释放出毁灭性的光之冲击！
    *   **物理攻击**：普通挥剑。
        *   *v12 新增说明*：使用光之剑进行普通的物理斩击。
*   **桃井 (随机策略)**：
    *   **盲盒模式**：随机执行普攻、给 Boss 挂 Debuff（降攻/降防）、给队友挂 Buff（治疗/增攻）。
    *   *v12 新增逻辑*：攻击增益会添加状态图标（⚔️）。
*   **小绿 (辅助治疗)**：
    *   **艺术润色**：稳定为全队回复生命值。
    *   *v12 新增逻辑*：治疗会添加状态图标（💚）。
*   **柚子 (强力输出/控制)**：
    *   **AI 逻辑**：主要进行普攻。
    *   **大招**：随机触发【通关指令·改】，直接眩晕 Boss 一回合。

## 3. 战斗数值与状态逻辑
*   **伤害计算**：`Max(1, 攻击 × 技能倍率 - 防御)`，并包含 ±5 的随机浮动。
*   **状态管理 (v12 增强)**：
    *   **Debuff/Buff**：基于回合数递减（如护盾持续 2 回合）。
    *   **状态图标**：支持在 UI 上显示 Buff/Debuff 图标（如 ⚔️, 💚, 🛡️）。
    *   **特殊状态**：支持【束缚】（Stunned）状态的传递与解除。
    *   **反射伤害**：在 `take_damage` 中实现了伤害反弹逻辑。
*   **死亡判定 (v11.4)**：
    *   `take_damage` 函数仅负责更新 HP 和返回伤害值。
    *   调用者（主程序）负责打印伤害信息，并在之后检查 `is_alive()` 状态。
    *   如果 `is_alive()` 返回 `False`，则打印死亡信息。

## 6. 交互式沙盒运行指南 (v15.3 新增)
由于沙盒环境的限制，无法直接通过标准输入获取用户指令（会导致 EOFError）。因此，在沙盒环境中运行模拟器时，需采用**交互式代码会话**模式。

### 6.1 运行步骤
1.  **启动会话**：使用 `start_code_session` 运行 `python main.py --monster <怪物名>` 或 `python main.py --level <等级>`。
    *   `--monster`: 指定特定怪物（支持名字或索引）。
    *   `--level`: 指定怪物总等级，系统会自动随机生成一组怪物。
2.  **读取输出**：使用 `read_code_output` 等待程序打印战斗日志和输入提示（如 `>>> 请输入指令：`）。
3.  **发送输入**：使用 `send_code_input` 发送指令（格式：`目标序号-技能序号`，例如 `1-1`）。
4.  **循环交互**：重复步骤 2 和 3，直到战斗结束（所有怪物或所有玩家倒下）。

### 6.2 示例流程
*   **指定怪物**：`python main.py --monster 优香大魔王`
    *   程序输出：显示战斗开始，爱丽丝行动提示。
    *   用户输入：`1-1` (爱丽丝对目标 1 使用技能 1：光啊！)
*   **指定等级**：`python main.py --level 50`
    *   程序输出：显示战斗开始，爱丽丝行动提示。
    *   用户输入：`1-1` (爱丽丝对目标 1 使用技能 1：光啊！)

### 6.3 注意事项
*   输入指令需严格遵循格式，否则程序会报错或执行默认操作。
*   若遇到 `EOFError`，说明程序正在等待输入，请检查是否遗漏了 `send_code_input` 步骤。

### 8.4 战术系统：动态站位与技能射程 (高优先级)

#### 8.4.1 战场拓扑结构 (Linear Queue)
*   **定义**：战场不再是二维网格，而是一条一维的**线性队列**。
*   **表示**：使用整数索引 `index` (0, 1, 2...) 来表示角色在场上的相对位置。
*   **初始状态**：游戏开始时，我方和敌方分别占据队列的前半段和后半段（或者交错排列，取决于开局设置）。
- [x] **已完成**：`main.py` 使用 `battle_field` 列表管理全场单位，所有实体均携带 `position` 属性。

#### 8.4.2 移动机制：挤压式换位 (Squeeze Move)
每个单位在每回合拥有**一次主动移动**的机会。

*   **移动规则**：
    *   单位可以指定任意一个合法的 `target_index`。
    *   **合法性判定**：目标位置必须在当前存活单位的数量范围内（即不能移出边界）。
*   **挤压算法 (The Squeeze Logic)**：
    *   当单位 `U` 从当前位置 `old_pos` 移动到 `new_pos` 时，中间的障碍物会发生位移。
    *   **向左移动 (`new_pos < old_pos`)**：位于 `[new_pos, old_pos - 1]` 区间内的所有其他单位，其索引全部 **+1**（向后推）。
    *   **向右移动 (`new_pos > old_pos`)**：位于 `[old_pos + 1, new_pos]` 区间内的所有其他单位，其索引全部 **-1**（向前拉）。
    *   **原地不动**：索引不变。
- [x] **已完成**：`utils.py` 中的 `squeeze_move` 函数已完美实现该逻辑，并在移动阶段调用。

#### 8.4.3 攻击机制：基于技能的索敌 (Skill-Based Targeting)
角色的攻击不再固定，而是完全依赖所使用的**技能射程**。

*   **技能定义 (`skill.py`)**：每个技能增加一个属性 `range` (int)。
*   **索敌流程**：确定施法者位置 -> 筛选有效目标 (`dist <= range`) -> 随机锁定。
*   **例外情况**：无目标技能（充能/AOE）无视射程限制。
- [x] **已完成**：`player.py` 与 `monster.py` 均已集成 `_find_valid_targets` 方法，严格校验射程后发起攻击。

#### 8.4.4 回合行动流程 (Turn Flow)
为了保证逻辑清晰，建议采用以下标准流程：
1.  **【移动阶段】**：玩家输入位置，执行挤压换位，刷新站位。
2.  **【行动阶段】**：选择技能，根据射程索敌并结算伤害/效果。
- [x] **已完成**：`main.py` 战斗循环已拆分为 `process_movement_phase` 与 `process_player_actions`/`process_monster_action`，流程严谨。
