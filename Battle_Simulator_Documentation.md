# ⚔️ 战斗模拟器 v44.3 架构与功能文档

## 1. 系统概览
这是一个基于 Python 开发的模块化回合制战斗模拟器。采用**主程序 + 数据驱动**的设计，支持多名玩家角色对抗具有独特机制的 Boss。
在 **v44.x 版本**中，系统进行了深度的战术重构，引入了**线性队列战场**、**物理挤压移动**、**全员能量系统**以及极具策略深度的**朝向与背刺机制**。

## 2. 核心模块架构

### 2.1 主程序 (`main.py`) - v44.3 版
*   **游戏流程**：初始化队伍 -> 随机抽取怪物 -> 进入战斗循环 -> 判定胜负。
*   **战斗循环**：
    1.  **【移动阶段】**：玩家输入位置，执行**无缝挤压换位**（v44.3 核心修复），刷新站位与朝向。
    2.  **【玩家行动阶段】**：依次执行队友（桃井、小绿、柚子）和主角（爱丽丝）的行动。支持技能射程索敌。
    3.  **【Boss 行动阶段】**：执行怪物逻辑（攻击、施法、受控）。怪物拥有独立的 AI 决策树，支持追击与智能索敌。
    4.  **【结算阶段】**：检查血量，处理 DoT 等持续状态，判定游戏结束。
*   **UI 反馈**：
    *   **可视化血条**：实时打印 HP 进度条。
    *   **状态图标**：显示 Buff/Debuff 图标（如 ⚔️, 💚, 🛡️, ☠️）。
    *   **朝向显示**：在状态栏显示单位面向 ➡️/⬅️。
    *   **战场概览**：每回合开始前及移动后，显示全场单位的阵营、位置和朝向。

### 2.2 技能模块 (`skill.py`) - v44.2 版
*   **技能注册表 (`SKILL_REGISTRY`)**：集中管理所有角色的技能实例。
*   **效果类体系**：
    *   `BaseSkillEffect`: 基类。
    *   `AttackEffect`: 单体/群体攻击。**集成了背刺判定**（若满足条件，伤害 x1.25）。
    *   `AoEAttackEffect`: 范围攻击。同样集成背刺判定。
    *   `HealEffect`, `BuffEffect`, `DebuffEffect`: 基础状态修改。
    *   `StunEffect`, `ImmobilizeEffect`: 强控效果。
    *   `DotEffect`: 持续伤害（中毒、出血、腐蚀等）。
    *   `BlindEffect`: 致盲（强制随机移动并跳过行动）。
    *   `SequenceEffect`: **连携技能**，允许一个技能包含多个子效果（如：攻击+上毒）。
    *   `CleanseEffect`: 清除增益，支持清空能量槽。
*   **背刺系统 (`_check_backstab`)**：全局辅助函数，根据相对位置和朝向判定是否触发背刺增伤。

### 2.3 玩家模块 (`player.py`) - v44.2 版
*   **爱丽丝 (手动交互)**：
    *   **光啊！赐予我力量！**：积攒能量（1次充能=1层）。
    *   **世界的法则即将崩坏！光哟！！！** (EX 技能)：消耗能量，造成巨额 AOE 伤害，并根据充能层数附加**致盲**效果。
    *   **光之剑，出鞘吧** (普攻)：复合技能，包含物理斩击和 30% 几率的**闪光致盲**。
*   **队友 (AI 自动)**：
    *   **桃井**：随机策略（普攻/Debuff/Buff）。
    *   **小绿**：稳定治疗（艺术润色）。
    *   **柚子**：普攻为主，随机触发眩晕控制 (Hit Stop!)。
*   **通用属性**：所有玩家单位均具备 `facing` (朝向) 属性，移动时会自动更新。

### 2.4 怪物模块 (`monster.py`) - v44.2 版
*   **数据结构**：包含 HP, ATK, DEF, Energy, Status Effects, Facing 等完整属性。
*   **AI 决策**：
    *   **智能索敌**：优先攻击射程内目标，避免对已有 Debuff 目标重复施加。
    *   **全速追击**：当目标脱战时，直接移动到最大射程边缘。
    *   **致盲反应**：致盲状态下强制进行随机乱撞。
*   **特色机制**：
    *   🕷️ **机械蜘蛛**：蛛网束缚 (Immobilize)。
    *   👻 **幽灵猫**：相位闪烁 (闪避下次攻击)。
    *   🐉 **古代巨龙**：龙息吐息 (AOE) + 灼烧 (DoT)。
    *   🌀 **虚空吞噬者**：视界崩坏 (Cleanse + 能量抽空) + 生命汲取 (Lifesteal)。
    *   🌀 **虚空吞噬者·终焉**：视界崩坏 + 深渊凝视 + 生命汲取 + 护盾 (强化版吸血与持续回复)。
    *   📐 **优香大魔王**：精确计算 (高倍率暴击)。

### 2.5 工具模块 (`utils.py`) - v44.3 版
*   **`squeeze_move` (物理挤压算法)**：
    *   **【v44.3 核心修复】**：废弃了导致索引漂移的 `pop/insert` 方案。
    *   **新方案 (全员平移)**：移动单位离开原位产生空缺 -> 中间单位向空位方向依次移位填补 -> 移动单位填入目标位。
    *   **特性**：实现了完美的无缝插队体验，同时自动更新移动者的朝向。

## 3. 战斗数值与状态逻辑

### 3.1 朝向系统 (Facing System)
*   **定义**：每个单位都有一个 `facing` 属性。
    *   `1`: 面向右 (正方向)。
    *   `-1`: 面向左 (负方向)。
*   **更新规则**：
    *   玩家/怪物向**右**移动 (Index 增加) -> 面向变为 `1`。
    *   玩家/怪物向**左**移动 (Index 减小) -> 面向变为 `-1`。
    *   原地不动 -> 朝向保持不变。

### 3.2 背刺机制 (Backstab System)
*   **触发条件**：
    1.  施法者位于目标**左侧** (Pos < Target Pos) 且 目标面向**右** (Facing == 1)。
    2.  施法者位于目标**右侧** (Pos > Target Pos) 且 目标面向**左** (Facing == -1)。
*   **效果**：伤害结果乘以 **1.25** 倍。
*   **应用范围**：所有 `AttackEffect`, `AoEAttackEffect`, `LifestealEffect`。

### 3.3 伤害计算
*   **公式**：`Max(1, (ATK × Multiplier ± Variance) - DEF)`
*   **修正因子**：暴击 (x1.5/x2.0)、背刺 (x1.25)、Debuff 减益。

### 3.4 状态管理
*   **Buff/Debuff**：回合结束时减少持续时间。
*   **DoT (持续伤害)**：在结算阶段触发，扣除 HP 并减少持续时间。
*   **特殊状态**：
    *   `is_blinded`: 本回合随机移动，无法主动行动。
    *   `is_immobilized`: 无法移动，但可以行动。
    *   `is_stunned`: 无法行动，无法移动。

#### 8.3.1 通用职业详细设计 (General Classes)
*(以下为主要通用职业的详细机制描述，供编码参考)*

**A. 重装卫士 (Heavy Guard)**
*   **定位**：绝对防御 / 团队护盾 / 阵地控制
*   **核心机制**：利用**朝向**和**挤压站位**。
*   **被动【钢铁壁垒】**：当重装卫士面向敌人（Front）时，他会形成一个保护领域。位于他**正后方**（Position - 1, Position - 2...）的队友受到的伤害减少 15%。如果重装卫士处于"嘲讽"状态，减伤提升至 30%。

### 8.4 战术系统：动态站位与技能射程 (高优先级)

#### 8.4.1 战场拓扑结构 (Linear Queue)
*   **定义**：战场不再是二维网格，而是一条一维的**线性队列**。
*   **表示**：使用整数索引 `index` (0, 1, 2...) 来表示角色在场上的相对位置。
*   **初始状态**：游戏开始时，我方和敌方分别占据队列的前半段和后半段（或者交错排列，取决于开局设置）。
- [x] **已完成**：`main.py` 使用 `battle_field` 列表管理全场单位，所有实体均携带 `position` 属性。

#### 8.4.2 移动机制：挤压式换位 (Squeeze Move)
每个单位在每回合拥有**一次主动移动**的机会。

*   **移动规则**：
    *   单位可以指定任意一个合法的 `target_index`。
    *   **合法性判定**：目标位置必须在当前存活单位的数量范围内（即不能移出边界）。
*   **挤压算法 (The Squeeze Logic)**：
    *   当单位 `U` 从当前位置 `old_pos` 移动到 `new_pos` 时，中间的障碍物会发生位移。
    *   **向左移动 (`new_pos < old_pos`)**：位于 `[new_pos, old_pos - 1]` 区间内的所有其他单位，其索引全部 **+1**（向后推）。
    *   **向右移动 (`new_pos > old_pos`)**：位于 `[old_pos + 1, new_pos]` 区间内的所有其他单位，其索引全部 **-1**（向前拉）。
    *   **原地不动**：索引不变。
- [x] **已完成**：`utils.py` 中的 `squeeze_move` 函数已完美实现该逻辑，并在移动阶段调用。

#### 8.4.3 攻击机制：基于技能的索敌 (Skill-Based Targeting)
角色的攻击不再固定，而是完全依赖所使用的**技能射程**。

*   **技能定义 (`skill.py`)**：每个技能增加一个属性 `range` (int)。
*   **索敌流程**：确定施法者位置 -> 筛选有效目标 (`dist <= range`) -> 随机锁定。
*   **例外情况**：无目标技能（充能/AOE）无视射程限制。
- [x] **已完成**：`player.py` 与 `monster.py` 均已集成 `_find_valid_targets` 方法，严格校验射程后发起攻击。

#### 8.4.4 回合行动流程 (Turn Flow)
为了保证逻辑清晰，建议采用以下标准流程：
1.  **【移动阶段】**：玩家输入位置，执行挤压换位，刷新站位。
2.  **【行动阶段】**：选择技能，根据射程索敌并结算伤害/效果。
- [x] **已完成**：`main.py` 战斗循环已拆分为 `process_movement_phase` 与 `process_player_actions`/`process_monster_action`，流程严谨。
